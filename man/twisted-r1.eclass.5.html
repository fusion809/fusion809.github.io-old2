NAME

twisted-r1.eclass - Eclass for Twisted packages

DESCRIPTION

The twisted eclass defines phase functions for Twisted packages.

FUNCTIONS

_twisted-r1_camelcase <pn>
Convert dash-separated <pn> to CamelCase name suitable for Twisted. In pure bash, therefore safe for global scope execution.
twisted-r1_python_test
The common python_test() implementation that suffices for Twisted packages.
python_test
Default python_test() for Twisted packages. If you need to override it, you can access the original implementation via twisted-r1_python_test.
twisted-r1_src_install
Default src_install() for Twisted packages. Automatically handles HTML docs (unless HTML_DOCS is set explicitly) and manpages in Twisted packages.
_twisted-r1_create_caches <packages>...
Create dropin.cache for plugins in specified packages. The packages are to be listed in standard dotted Python syntax.
twisted-r1_update_plugin_cache
Update and clean up plugin caches for packages listed in TWISTED_PLUGINS.
twisted-r1_pkg_postinst
Post-installation hook for twisted-r1. Updates plugin caches.
twisted-r1_pkg_postrm
Post-removal hook for twisted-r1. Updates plugin caches.

ECLASS VARIABLES

TWISTED_PN ?= "$(_twisted-r1_camelcase ${PN})"
The real package name. Default to camel-case conversion of ${PN}.
 
Example: TwistedCore
TWISTED_P ?= ${TWISTED_PN}-${PV}
The real package name with version appended.
 
It is used to build the default SRC_URI and S values.
 
Example: TwistedCore-1.2.3
TWISTED_RELEASE ?= "$(get_version_component_range 1-2 ${PV})"
The 'release' of Twisted. Defaults to the major & minor version number from ${PV}.
 
It is used to build the default SRC_URI. It may be also used in dependencies against other Twisted packages.
 
Example: 1.2
TWISTED_PLUGINS = ( twisted.plugins )
An array of Twisted plugins, whose cache is regenerated in pkg_postinst() and pkg_postrm() phases.
 
If no plugins are installed, set to empty array.
