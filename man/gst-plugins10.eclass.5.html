NAME

gst-plugins10.eclass - Manages build for invididual ebuild for gst-plugins.

DESCRIPTION

Eclass to make external gst-plugins emergable on a per-plugin basis and to solve the problem with gst-plugins generating far too much unneeded dependancies.
 
GStreamer consuming applications should depend on the specific plugins they need as defined in their source code.
 
In case of spider usage, obtain recommended plugins to use from Gentoo developers responsible for gstreamer <gstreamer@gentoo.org> or the application developer.

FUNCTIONS

gst-plugins10_system_link gst-plugins10_system_link gst-libs/gst/audio:gstreamer-audio [...]
Walks through makefiles in order to make sure build will link against system librairies. Takes a list of path fragments and corresponding pkgconfig libraries separated by colon (:). Will replace the path fragment by the output of pkgconfig.
gst-plugins10_src_configure
Handles logic common to configuring gstreamer plugins
gst-plugins10_src_compile
Compiles requested gstreamer plugin.
gst-plugins10_src_install
Installs requested gstreamer plugin.

ECLASS VARIABLES

GST_LA_PUNT
Should we delete all the .la files? NOT to be used without due consideration. Defaults to no for EAPI < 5.
GST_PLUGINS_BUILD ?= ${PN/gst-plugins-/}
Defines the plugins to be built. May be set by an ebuild and contain more than one indentifier, space seperated (only src_configure can handle mutiple plugins at this time).
GST_PLUGINS_BUILD_DIR ?= ${PN/gst-plugins-/}
Actual build directory of the plugin. Most often the same as the configure switch name.
GST_TARBALL_SUFFIX
Most projects hosted on gstreamer.freedesktop.org mirrors provide tarballs as tar.bz2 or tar.xz. This eclass defaults to bz2 for EAPI 0, 1, 2, 3 and defaults to xz for everything else. This is because the gstreamer mirrors are moving to only have xz tarballs for new releases.
GST_ORG_MODULE ?= $PN
Name of the module as hosted on gstreamer.freedesktop.org mirrors. Leave unset if package name matches module name.
