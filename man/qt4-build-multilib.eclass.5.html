NAME

qt4-build-multilib.eclass - Eclass for Qt4 split ebuilds with multilib support.

DESCRIPTION

This eclass contains various functions that are used when building Qt4. Requires EAPI 5.

FUNCTIONS

qt4-build-multilib_src_unpack
Unpacks the sources.
qt4-build-multilib_src_prepare
Prepare the sources before the configure phase. Strip CFLAGS if necessary, and fix the build system in order to respect CFLAGS/CXXFLAGS/LDFLAGS specified in make.conf.
qt4-build-multilib_pkg_postinst
Regenerate configuration after installation or upgrade/downgrade.
qt4-build-multilib_pkg_postrm
Regenerate configuration when a module is completely removed.
qt_use <flag> [feature] [enableval]
<flag> is the name of a flag in IUSE.
 
Outputs "-${enableval}-${feature}" if <flag> is enabled, "-no-${feature}" otherwise. If [feature] is not specified, <flag> is used in its place. If [enableval] is not specified, the "-${enableval}" prefix is omitted.
qt_native_use <flag> [feature] [enableval]
<flag> is the name of a flag in IUSE.
 
Outputs "-${enableval}-${feature}" if <flag> is enabled and we are currently building for the native ABI, "-no-${feature}" otherwise. If [feature] is not specified, <flag> is used in its place. If [enableval] is not specified, the "-${enableval}" prefix is omitted.
qt4_symlink_framework_headers
On OS X we need to add some symlinks when frameworks are being used, to avoid complications with some more or less stupid packages.

ECLASS VARIABLES

PATCHES
Array variable containing all the patches to be applied. This variable is expected to be defined in the global scope of ebuilds. Make sure to specify the full path. This variable is used in src_prepare phase.
 
Example:
PATCHES=(
"${FILESDIR}/mypatch.patch"
"${FILESDIR}/mypatch2.patch"
)
QT4_TARGET_DIRECTORIES
Space-separated list of directories that will be configured, compiled, and installed. All paths must be relative to ${S}.
QCONFIG_ADD
List of options that must be added to QT_CONFIG in qconfig.pri
QCONFIG_REMOVE
List of options that must be removed from QT_CONFIG in qconfig.pri
QCONFIG_DEFINE
List of macros that must be defined in QtCore/qconfig.h
