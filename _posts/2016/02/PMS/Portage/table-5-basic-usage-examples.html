Basic Usage

Table 5: Basic Usage of Portage
Action Command1 Description
Build package from source {% include Code/coder.html line1="emerge -B $package" %} Building packages from source code is as simple as passing the necessary flag to emerge.
Install build dependencies {% include Code/coder.html line1="emerge -o $package" %} Install the required build dependencies for $package.
Install package {% include Code/coder.html line1="emerge $package" %} Install the target package ($package) and all runtime dependencies.
Remove package {% include Code/coder.html line1="emerge -C $package" %} Remove the target package ($package) and all runtime dependencies not required by other installed packages.
Search for keyword {% include Code/coder.html line1="emerge --search $package" %} Search package names and descriptions for provided regex.
Upgrade all packages {% include Code/coder.html line1="emerge --sync" line2="layman -S" line3="emerge -uDU --with-bdeps=y @world" %} Upgrade all installed packages, after synchronizing (with their respective version control system) the Portage Tree and all locally-installed overlays.
1: Note: in this table I use the short-hand notation $package to refer to the target package(s) that the user wishes to perform the mentioned action on.
